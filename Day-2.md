# Day 2: Code the first and second specification

[← Back to README](./README.md) · [← Prev: Day 1](./Day-1.md) · [Next: Day 3 →](./Day-3.md)

## Session 1: Deeper dive on Plan, Clarify, Tasks, and Implement

**Narrative**: The SE Dev Crew will walk through the `Plan -> Clarify -> Tasks -> Implement` steps of the project.

- **Led by**: SE Dev Crew
- **Required participants**: Technical specialists and coders
- **Goal**: Define the plans for each of the specifications
- **Output**: Necessary inputs to the plans for each of the three (3) specifications

## Session 2: Run `specify` for the first specification

**Narrative**: Run the `specify` and `clarify` steps to establish the first specification. The SE team along with the customer technical specialists and devs will together establish the detailed plans for the first specification. This is done by running the `plan`, `clarify`, and `tasks` commands from SpecKit.

- **Led by**: SE and customer dev teams
- **Required participants**: Technical specialists and coders
- **Goal**: Create the plans and tasks for the first specification
- **Output**: Fully established specification and tasks for the coding agent to execute

## Session 3: Execute implementation of the first specification

**Narrative**: The SE and customer dev teams execute the implementation of the first specification. They ensure results are pushed to the Git repo and work collaboratively to troubleshoot and clean up any code that is not working correctly.

- **Led by**: SE and customer dev teams
- **Required participants**: Technical specialists and coders
- **Goal**: Implement the first specification
- **Output**: Fully functional implementation of the first specification — committed to GitHub

## Session 4: Run `specify` for the second specification

**Narrative**: Run the `specify` and `clarify` steps to establish the second specification. The SE team along with the customer technical specialists and devs will together establish the detailed plans for the second specification. This is done by running the `plan`, `clarify`, and `tasks` commands from SpecKit.

- **Led by**: SE and customer dev teams
- **Required participants**: Technical specialists and coders
- **Goal**: Implement the second specification
- **Output**: Fully functional implementation of the second specification — committed to GitHub
